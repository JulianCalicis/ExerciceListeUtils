# 🛡️ Projet Sécurisé™ (Ou presque)

Bienvenue sur ce magnifique repository ! Un espace de liberté absolue où le chaos est roi et où la notion de "branche protégée" est considérée comme un concept purement théorique, voire une légende urbaine.

## 🚀 Fonctionnalités Majeures

* Intégration Continue (CI) Manuelle : Pourquoi s'embêter avec des pipelines automatiques quand on peut tester son code directement en production ?
* Sécurité Décentralisée : La sécurité repose entièrement sur la confiance aveugle envers chaque personne ayant un accès Internet.
* Historique Git Alternatif : Des commits comme "test2", "fix final final", ou "oups" directement sur main. Un vrai roman policier.

---

## 🛑 Guide d'utilisation (Pour les invités)

Si tu passes par là et que tu as envie de t'amuser, n'hésite pas :

1. Fais tes modifications directement sur la branche main.
2. Ne crée surtout pas de Pull Request (ce serait une perte de temps).
3. Lance un bon vieux :
   git push origin main --force
4. Regarde le monde brûler. 🔥

## ⚠️ Message à l'attention du propriétaire du repo

Dis, mon cher ami... On t'a déjà parlé des "Protected Branches" sur GitHub/GitLab ? 

Parce que là, n'importe qui (moi y compris, la preuve) peut venir écraser ton travail, glisser un script qui mine du Dogecoin, ou simplement supprimer tout ton code source sur un coup de tête après trois bières.

### 🛠️ Comment réparer ce désastre en 30 secondes :

1. Va dans les Settings de ton repository.
2. Clique sur Branches.
3. Ajoute une règle de protection sur main / master.
4. Coche "Require a pull request before merging".
5. Respire, ton code est enfin à l'abri de mes bêtises.
